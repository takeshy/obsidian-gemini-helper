# Workflow Nodes

Workflow YAML is usually generated by AI, but users may ask what node to use or how nodes behave. Each node has an `id`, a `type`, type-specific properties, and usually `next` to continue execution.

# Node Categories

- Variables: `variable`, `set` declare and update variables.
- Control: `if`, `while` branch or loop. Conditional edges use `true` and `false` labels.
- LLM: `command` runs a Gemini prompt and saves text or generated image output.
- Data: `http`, `json`, `script` call APIs, parse JSON, or run JavaScript.
- Notes: `note`, `note-read`, `note-search`, `note-list`, `folder-list`, `open` operate on vault content.
- Files: `file-explorer`, `file-save` select or save binary/text files.
- Prompts: `prompt-file`, `prompt-selection`, `dialog` ask the user for file, selection, or dialog input.
- Composition: `workflow` runs another workflow as a sub-workflow.
- RAG: `rag-sync` syncs notes to the selected RAG store.
- External: `mcp`, `obsidian-command` call MCP tools or Obsidian commands.
- Utility: `sleep` pauses execution.

# Key Nodes

`command` supports `prompt`, optional `model`, `ragSetting` (`__websearch__`, `__none__`, or setting name), `vaultTools` (`all`, `noSearch`, `none`), `mcpServers`, `attachments`, `enableThinking`, `saveTo`, and `saveImageTo`.

`note` writes content with `path`, `content`, `mode` (`overwrite`, `append`, `create`), `confirm`, and `history`. Confirmation defaults to safe behavior.

`note-read` reads a file into `saveTo`; encrypted files trigger a password prompt and cache the password for the session.

`note-list` can filter by folder, recursion, tags, creation or modification window, sort field, sort order, and limit.

`note-search` searches file names or contents and returns matched context.

`folder-list` lists vault folders from a parent folder.

`open` opens a vault file in Obsidian.

`dialog` shows a modal dialog, can render Markdown, can collect text input, and can return selected buttons or options.

`prompt-file` asks the user to select a Markdown file. In hotkey and event runs it can automatically use the active or event file.

`prompt-selection` asks for editor selection. In hotkey runs it uses the current selection or full active file content; in event runs it uses the event file content.

`file-explorer` selects any supported file and returns `FileExplorerData` with path, name, extension, MIME type, content type, and text or Base64 data.

`file-save` saves FileExplorerData or generated binary/text data back to the vault.

`workflow` executes another workflow as a sub-workflow and can pass variables.

`rag-sync` syncs eligible vault content to the selected RAG store.

`http` sends external requests. Warn users that HTTP nodes can transmit vault data to third-party services.

`json` expects `source` to be a bare variable name, not `{{source}}` interpolation.

`script` runs JavaScript in the workflow sandbox.

`mcp` calls an enabled MCP server tool configured in plugin settings.

`obsidian-command` runs an Obsidian command by command ID and is useful for operations such as encryption workflows.

`sleep` pauses execution for a configured duration.

# Workflow Options

`options.showProgress: false` hides the execution progress modal for hotkey or workflow-list execution. The visual workflow panel always shows progress.

# Skill Workflow Outputs

When invoked from chat through `run_skill_workflow`, every variable whose name does not start with `_` is returned to the chat model. A final `command` node just to display output is unnecessary; put output presentation rules in `SKILL.md`.
